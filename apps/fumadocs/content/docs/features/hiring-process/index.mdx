## Shared Decisions

| Decision                           | Why                                                             |
| ---------------------------------- | --------------------------------------------------------------- |
| Domain constants pattern for enums | Type-safe values shared between domain, database, and UI layers |
| Zod schemas as source of truth     | Single definition drives validation, types, and API contracts   |
| Eden Treaty for API type safety    | End-to-end type safety from Elysia backend to React frontend    |

## Shared Gotchas

- Database enums must use domain constant values arrays (`SALARY_RATE_TYPE_VALUES`) to stay in sync.
- Form default values must match database defaults for backward compatibility when adding new fields.
